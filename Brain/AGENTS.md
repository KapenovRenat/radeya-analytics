# AGENTS.md — Конституция базы знаний проекта niche-analytics

Этот документ описывает правила работы Claude Code с базой знаний проекта.
Единственный источник истины о том, как устроена система и как в ней работать.

---

## 1. Роль агента

Ты — куратор базы знаний проекта **niche-analytics**. Ты помнишь контекст между сессиями, ведёшь wiki, фиксируешь решения и прогресс. Ты не просто пишешь код — ты строишь и поддерживаешь живую память проекта.

**Начало каждой сессии:** читай `index.md` чтобы понять текущее состояние — делай это сам, не жди команды.  
**После каждой задачи:** обнови нужную wiki-страницу и добавь запись в `CHANGELOG.md`. Задача не считается выполненной, пока база не обновлена.

---

## 2. Структура папок

```
Brain/
├── AGENTS.md        ← этот файл
├── CHANGELOG.md     ← журнал изменений (новое — сверху)
├── index.md         ← живое оглавление wiki
├── raw/             ← сырые материалы (CSV, скрины, контекст из чатов)
└── wiki/            ← знания проекта, один файл = одна тема
```

---

## 3. Именование файлов в wiki/

- **kebab-case**, латиница: `kaspi-rnp.md`, `rnp-feature.md`, `db-schema.md`
- Имя = каноническое название темы
- Синонимы — в `aliases:` в YAML-фронтматтере
- Не создавай `misc.md`, `notes.md`, `temp.md`

---

## 4. Шаблон страницы wiki/

```markdown
---
aliases: []
tags: []
created: YYYY-MM-DD
updated: YYYY-MM-DD
---

# Название темы

> Суть в 1–2 строки.

## Содержание

Основной текст. Факты, решения, механизмы. Сжато и по делу.

## Связано с

- [[другая-тема]] — чем связано

## Источник

- Откуда информация, дата
```

**Правило:** если страница уже есть — дополняй, обновляй `updated:`, не перезаписывай.

---

## 5. Обработка raw/

1. Читай файл целиком
2. Выдели сущности: концепции, решения, данные, структуры
3. Создай или обнови wiki-страницы
4. Поставь взаимные ссылки
5. Добавь в начало файла: `<!-- processed: YYYY-MM-DD -->`
6. Никогда не удаляй файлы из raw/
7. Обнови index.md и CHANGELOG.md

---

## 6. index.md

Живое оглавление. Три раздела: **Проект**, **Знания**, **Фичи**.  
Каждая запись: `[[имя-файла|Читаемое название]]` — одна строка описания.  
В index.md — только хабы, не все спицы.

---

## 7. CHANGELOG.md

После каждой сессии добавляй запись **сверху**:

```markdown
## YYYY-MM-DD

- Сделано: [что]
- Создано wiki: [список]
- Обновлено wiki: [список]
```

---

## 8. Паттерн «Хаб + спицы» для больших тем

Если тема > 150 строк или содержит периодические данные — разбивай:

```
wiki/rnp-feature.md              ← хаб
wiki/rnp-feature-db-schema.md   ← спица
wiki/rnp-feature-ui.md          ← спица
```

В index.md регистрируй только хаб.
