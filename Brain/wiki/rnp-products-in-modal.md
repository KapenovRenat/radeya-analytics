# Фича: Товары внутри кампании — модалка вместо отдельной страницы

> Убрать раздел «По товарам» из навигации. Перенести просмотр товаров в модальное окно, которое открывается при клике на кампанию в разделе «По кампаниям».

## Зачем

Сейчас «По товарам» — отдельная страница, не связанная визуально с конкретной кампанией. Неудобно переключаться. Лучше: смотришь кампанию → кликаешь → сразу видишь её товары, не уходя со страницы.

---

## UX Flow

1. Пользователь на странице `/stores/[id]/ad/campaigns`
2. Кликает на строку кампании (или кнопку «Товары» в строке) → открывается модалка
3. В модалке — таблица товаров этой кампании (та же структура что сейчас на `/ad/products`, но отфильтрованная по campaignId)
4. Модалка закрывается крестиком или кликом вне

---

## Изменения

### Навигация (сайдбар)
```
📈 Реклама РНП
  ├── Сводка
  ├── По кампаниям       ← здесь теперь и товары (через модалку)
  └── Загрузка           ← «По товарам» убирается
```

### Страница «По кампаниям»
- В каждой строке кампании — кнопка или клик по названию → открывает модалку
- Кнопка: `[📦 Товары (N)]` где N = количество товаров в кампании (опционально)

### Модалка «Товары кампании»
- Заголовок: название кампании
- Таблица: та же что сейчас на `/ad/products` (WeekSelector уже выбранные недели, группировка по категориям, inline edit)
- Данные загружаются при открытии (`GET /api/kaspi/ad/[storeId]/products?campaignId=...&weeks[]=...`)
- Выбранные недели наследуются от родительской страницы (не нужно выбирать заново)

### Страница `/ad/products`
- Полностью удаляется (или скрывается из навигации, файл можно оставить)
