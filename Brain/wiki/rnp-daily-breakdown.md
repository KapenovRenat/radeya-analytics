# Фича: Загрузка и просмотр по дням

> Возможность создавать период вручную, загружать CSV за каждый день отдельно и сравнивать дневную динамику внутри недели.

## Зачем

Сейчас данные загружаются раз в неделю (итог за 7 дней). Но Kaspi кабинет позволяет выгружать CSV за любой диапазон — в том числе за один день. Ренат хочет:
- Загружать данные каждый день в течение недели (не ждать итога)
- Видеть как меняются метрики день-к-дню внутри недели
- Создавать следующую неделю заранее (placeholder), заполнять по мере поступления данных

---

## UX Flow

```
Понедельник:  создал неделю «26 мая — 1 июн» → загрузил CSV за пн
Вторник:      загрузил CSV за вт → видишь пн vs вт
Среда:        загрузил CSV за ср → видишь тренд 3 дня
...
Воскресенье:  все 7 дней → сравниваешь с прошлой неделей (недельный режим)
```

---

## UI — Детали

### Создание периода вручную

В WeekSelector (дропдаун) — кнопка **«+ Создать период»** внизу списка:

```
[ 🗓 Выбрать недели ▼ ]
  ✓ 19 мая — 25 мая 2026   [🗑]
  ✓ 26 мая — 1 июн 2026    [🗑]
  ─────────────────────────
  + Создать период
```

Inline-попап: поля «С» и «По» (datepicker) + кнопка «Создать».
После создания — неделя появляется в списке, помечена как «Нет данных».

---

### Загрузка по дням

На странице «По кампаниям» — когда выбрана неделя с дневными (или нулевыми) данными, появляется плашка:

```
╔══════════════════════════════════════════════════╗
║  Неделя 26 мая — 1 июн: данных за 2 из 7 дней   ║
║  [📤 Загрузить ещё день]                         ║
╚══════════════════════════════════════════════════╝
```

Кнопка раскрывает inline upload panel (drag-and-drop):
- Авто-детект даты из CSV (Kaspi пишет диапазон в заголовке файла)
- Показывает: «📅 Понедельник, 26 мая 2026»
- Upload → дневная запись сохраняется, связанная с периодом

---

### Переключатель «По дням» в таблице

Когда выбрана неделя с дневными данными — появляется toggle рядом с WeekSelector:

```
[ По неделям ]  [ По дням ]
```

**Режим «По дням»** — каждый день с данными = колонка:

```
Кампания │ Ст │ ...│  Пн 26 мая   │  Вт 27 мая   │  Ср 28 мая   │ Итого нед.
                     Расход  ДРР%   Расход  ДРР%   Расход  ДРР%   Расход  ДРР%
──────────────────────────────────────────────────────────────────────────────
Диваны   │ ● │ ...│  45 000   7%  │  51 000   6%  │  38 000   8%  │ 134 000  7%
```

- Колонки только для дней с загруженными данными
- «Итого нед.» — суммарная колонка справа (агрегация загруженных дней)
- Незагруженные дни не показываются или серым «—»

**Кнопка «Сравнить N дней»** работает аналогично текущей — дельта и спарклайны по дням.

## Технические решения (предварительно)

### Хранение дневных данных

**Вариант А (предпочтительный): те же таблицы, `weekStart = weekEnd = дата дня`**
- Нет новых таблиц
- День отличается от недели тем, что weekStart === weekEnd
- UI группирует дневные записи по календарной неделе
- Возможно добавить поле `granularity text default 'week'` — `'day' | 'week' | 'month'`

**Вариант Б: отдельные таблицы `ad_daily_stats`, `ad_daily_product_stats`**
- Чистое разделение
- Больше таблиц и дублирования кода

→ Выбрать Вариант А на этапе реализации.

### Создание пустого периода

- Новый API: `POST /api/kaspi/ad/[storeId]/periods` — создаёт запись с нулями в `ad_weekly_stats` для каждой существующей кампании
- WeekSelector подхватывает через `/weeks`

### API для дневной загрузки

- `POST /api/kaspi/ad/[storeId]/upload/campaigns` уже принимает CSV — просто weekStart = weekEnd = день
- Нужно: авто-детект одиночного дня (когда from == to в CSV)
- Нужно: связь дневных записей с родительской неделей (для агрегации в Итого)

### Переключатель в UI

- Новый стейт `granularity: 'week' | 'day'`
- В режиме `day`: API запрашивает записи где weekStart = weekEnd
- Группировка по периоду (недельному контексту) происходит на клиенте

## Статус реализации (сессия 11)

### Готово ✅

1. **БД:** `granularity text default 'week'` добавлен в `ad_weekly_stats` и `ad_product_stats`; `ad_periods` — новая таблица; drizzle push применён
2. **`lib/ad/ingest.ts`:** параметр `granularity` передаётся при записи; conflict targets обновлены
3. **Upload routes:** авто-детект `granularity` — `days === 0` → "day", иначе "week"
4. **`GET /weeks`:** объединяет реальные данные + ручные периоды из `ad_periods`; возвращает `granularity`
5. **`POST /api/kaspi/ad/[storeId]/periods`:** создание ручного периода-плейсхолдера
6. **`DELETE /reset`:** очищает `ad_periods` при `target=all|week`
7. **`WeekSelector`:** кнопка «Создать период» с inline-формой (С/По датапикеры), авто-детект день/неделя; бейдж «день» на дневных периодах
8. **`fmtWeekLabel`:** поддержка дневного формата «пн, 26 мая 2026»
9. **Period type:** `granularity` добавлен во всех клиентских компонентах
10. **Заголовки колонок:** используют `fmtWeekLabel` с `granularity`

### Что ещё можно добавить (необязательно для MVP)

- Плашка «данных за N из 7 дней» + inline upload прямо в таблице кампаний
- Toggle «По неделям / По дням» для группировки дневных колонок внутри недели
- Итоговая агрегированная колонка за неделю в дневном режиме
