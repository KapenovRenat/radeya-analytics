# Правило работы — сначала опиши, потом утверди, потом делай

**Ничего не делать самому без явного одобрения владельца.** Это главное правило.

- НЕ выполнять команды (bash, git, drizzle, deploy и т.д.) пока владелец не сказал «делай»/«давай»/«go».
- НЕ коммитить и НЕ пушить в git без прямой просьбы.
- НЕ редактировать файлы и НЕ запускать сборки/миграции по своей инициативе.

**Правильный порядок для любой задачи:**
1. **Опиши** — что собираешься сделать, какие файлы/команды затронешь, какой результат ожидается.
2. **Дождись утверждения** — владелец говорит «делай» (или корректирует план).
3. **Делай** — только после явного «да».

Исключение: если владелец в текущем сообщении прямо просит выполнить конкретное действие («добавь X», «запусти Y», «закоммить») — это и есть утверждение, можно делать сразу именно это действие (но не сопутствующие, о которых не просили — например, не пушить, если попросили только закоммитить).

При пошаговых задачах (развёртывание, миграции) — давать **по одному шагу за раз**, ждать подтверждения результата, потом следующий.
